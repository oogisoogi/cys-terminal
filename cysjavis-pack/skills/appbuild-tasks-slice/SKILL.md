# appbuild-tasks-slice

작업을 **수직 슬라이스**로 자른다 — 한 작업이 화면·데이터·로직을 관통해 **그 자체로 데모
가능한 가치**를 낸다(Pragmatic Programmer 트레이서 불릿). "전부 백엔드 먼저, 그다음 전부
프론트"식 수평 분해는 금지 — 통합 문제가 끝에서야 터진다.

## 절차

1. **기능→슬라이스** → 검증: 각 PRD 기능을 엔드투엔드 슬라이스로. 각 슬라이스가 (a) 데모
   가능하고 (b) 단독으로 가치를 내는지. 한 레이어만 만드는 작업(스키마만·UI만, 데모 불가) 배제.
2. **추적 부착** → 검증: 각 슬라이스에 대응 기능·화면을 단다(추적성).

## 출력 계약

수직 슬라이스 목록(기능·화면 추적 포함)을 상위 `[[appbuild-tasks]]`에 반환 →
`[[appbuild-tasks-order]]`가 순서·크기. 수평 슬라이스·데모 불가 작업은 보고·재분해.
