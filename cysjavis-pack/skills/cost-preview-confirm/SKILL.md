# cost-preview-confirm

영상 파이프라인의 유료 단계(아바타 렌더·음성 합성·이미지/영상 생성·업스케일)는 전부
**종량제**다 — HeyGen 분당 과금, fal 초당/장당, ElevenLabs 문자당. 무심코 큰 작업을 돌리면
비용이 샌다. 그래서 **돈 나가는 호출 전에 항상 예상 비용을 보여주고 확인을 받는다.**

이 규약은 cysjavis가 강력 command를 승인 게이트로 다루는 방식과 같은 결 — 다만 여기선
*비용*이 게이트다.

## 규약 (모든 유료 외부 호출 스킬이 따른다)

1. **실행 전 추산** → 검증: 호출 직전 예상 비용을 산출한다.
   - HeyGen: 분량(초) × 해상도 단가(예: 1080p ~$4/분 — 지표값, 실단가는 플랜/응답 기준).
   - ElevenLabs: 문자 수 × 모델 단가(또는 쿼터 차감량).
   - fal: 모델별 장당/초당 단가 × 수량·길이(가능하면 응답의 가격 필드, 없으면 모델 문서값).
2. **표시** → 검증: `[작업 · 수량/길이 · 모델 · 예상 $X · 누적 세션 $Y]`를 한 줄로 보여준다.
   추산 근거(단가 출처)도 명시 — 추측 단가면 "지표값"이라 표시.
3. **확인** → 검증: 사용자(또는 오케스트레이터 정책)의 진행 확인을 받는다. 임계액
   (예: 단건 $1 또는 세션 $10) 초과면 명시적 확인 필수, 이하면 자동 진행 가능(정책화).
4. **누적 추적** → 검증: 세션 누적 비용을 기록(`run-cost.json`)해 폭주를 조기에 잡는다.
5. **사후 실측** → 검증: 응답에 실제 과금이 오면 추산과 비교해 기록(다음 추산 보정).

## 적용 대상

`[[heygen-avatar-render]]`(분당) · `[[voice-clone-elevenlabs]]`(문자당) ·
`[[media-gen]]` 계열(장/초당) · `[[audio-post]]`(생성 음악) · 업스케일 등 모든 유료 단계.
무료·로컬 단계(`[[video-stitch]]`·`[[video-verify]]`·HyperFrames 렌더)는 비대상.

## 출력 계약

`run-cost.json` — `[{step, model, qty, est_usd, actual_usd, ts}]` 누적. 임계 초과 미확인
호출은 차단. 오케스트레이터는 종료 보고에 세션 총비용을 포함한다.
