# appbuild-plan-quick

아이디어가 이미 명확하거나 사용자가 숙련됐을 때 — 긴 인터뷰·토론 없이 **핵심 결정만** 빠르게
받고 산출물을 즉시 작성한다.

## 절차

1. **핵심 결정 수집** → 검증: 꼭 필요한 것만 한 번에 묻는다 — 기능 목록·MVP 범위·스택·
   데이터 저장·인증 유무. 사용자가 한 번에 답.
2. **즉시 작성** → 검증: 답으로 `01-prd.md`·`03-architecture.md`를 바로 작성. 단 **수용 기준·
   데이터 모델은 생략하지 않는다**(뒤 감독관 추적성 검증의 토대 — 빠지면 게이트에서 막힌다).
3. **빠른 확인** → 검증: 산출물을 1회 제시·확인. 보강 필요하면 인터뷰/디베이트로 격상 안내.

## 출력 계약

`.appbuild/01-prd.md`·`.appbuild/03-architecture.md`(수용 기준·데이터 모델 포함). 상위
`[[appbuild-plan]]`로 반환. 퀵이라도 추적성에 필요한 최소 골격은 반드시 채운다.
