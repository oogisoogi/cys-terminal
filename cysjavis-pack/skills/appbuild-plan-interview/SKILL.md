# appbuild-plan-interview

사용자의 머릿속 의도를 질문으로 끌어내 빠짐없는 요구로 만든다. 한 번에 다 묻지 않고
**분기마다 합의**에 이를 때까지 캐묻는다 — `[[grill-me]]`(가정 명시·분기 질문·모서리 사냥·
합의 선언)와 `[[grill-with-docs]]`(도메인 언어 대조)의 방법을 그대로 쓴다.

## 절차

1. **의도 청취** → 검증: 사용자의 한 줄 아이디어를 받는다(예: "일정 만들고 캘린더에서 보는 MVP").
2. **분기 인터뷰** → 검증: `[[grill-me]]`로 모서리를 사냥한다 — 핵심 기능·범위 경계·데이터·
   인증 유무·에러/빈 상태·완료 기준. 모호한 분기마다 합의 선언까지 질문 반복.
3. **요구 정리** → 검증: 합의된 요구를 기능 목록(각 수용 기준)으로 정리. 사용자 최종 확인.

## 출력 계약

합의된 요구 묶음을 상위 `[[appbuild-plan]]`에 반환 → `01-prd.md`·`03-architecture.md`로 기록.
미합의 분기가 남으면 멈추고 추가 질문(빌드로 넘기지 않는다).
