# appbuild-screen-spec-flow

기능을 화면에 배치하고 화면 간 길을 놓는다(IA·내비게이션). 빌드의 골격이다.

## 절차

1. **화면 도출** → 검증: `01-prd.md`의 기능을 수행할 화면 목록을 만든다. **모든 기능이 최소
   한 화면에 매핑**되는지(고아 기능 0).
2. **흐름·진입점** → 검증: 화면 간 이동(내비게이션)·초기 진입점·주요 사용자 경로를 정의.
   막다른 화면(빠져나갈 길 없음)·도달 불가 화면 없게.
3. **매핑 표** → 검증: `기능 → 화면` 매핑 표를 만든다(감독관 추적성 검증의 입력).

## 출력 계약

화면 목록 + 내비게이션 + `기능↔화면` 매핑을 상위 `[[appbuild-screen-spec]]`에 반환 →
`[[appbuild-screen-spec-detail]]`이 각 화면을 상세화. 고아 기능·고립 화면은 보고·보강.
