# appbuild-supervisor-gate

"무엇을 충족하면 끝인가"를 스펙에서 **파생**한다 — 사람이 즉흥으로가 아니라. 산출물
`.appbuild/05-gate.md`. 핵심: **증거 기반 완료** — 게이트 행은 테스트 ID와 PASS 출력이 붙어야
초록, "다 됐다"는 자기 주장은 불인정(DoD=전역 증거 게이트, 수용기준=기능별 조건).

## 게이트 문서 구조 (파생 템플릿)

```markdown
# COMPLETION GATE — <project> (01~04에서 파생)

## A. 기능별 수용 (PRD 요구당 1행)
| Req | 수용기준(EARS WHEN…SHALL…) | 화면 | 데이터 | 작업 | E2E 테스트ID | 증거(PASS 로그) |
가게이트: 전 행 채움·빈칸 0·E2E PASS(출력 첨부)·커버리지%=100.

## B. 전역 DoD (모든 기능)
- [ ] 코드 리뷰 · 단위+통합+E2E 테스트 작성 AND PASS(출력 첨부, 주장 아님)
- [ ] 수용 기준 충족 · 비-해피(로딩/빈/에러) 검증 · 인증/권한 명세대로 · 입력검증·엣지
- [ ] critical/high 결함 0 · 문서 갱신 · 스테이징 배포·데모 가능

## C. 비기능 게이트 (측정·수치)
- [ ] 성능 <지표>≤<임계> 측정값:__ · 보안 검증:__ · 가용/확장 목표 측정:__

## D. 교차정합 (13항목 클린)
- [ ] 고아 요구·작업 0(RTM 양방향 클린) · CRITICAL/HIGH 0 · 미해결 placeholder 0

GATE 결과: A=100% · B 전부 체크(증거) · C 측정 · D 클린일 때만 PASS.
```

## 절차

1. **기능별 수용 파생** → 검증: 각 PRD 요구 → RTM 한 행(수용기준·화면·데이터·작업·E2E ID).
2. **전역 DoD·비기능** → 검증: 프로젝트에 맞는 DoD·비기능 임계(수치)를 채운다.
3. **증거 슬롯** → 검증: 각 행에 "증거(테스트 PASS 출력)" 칸을 비워 둔다 — 빌드가 채운다.

## 출력 계약

`.appbuild/05-gate.md`(증거 칸 비어 있음). 상위 `[[appbuild-supervisor]]`로 반환 →
`[[appbuild-orchestrate]]`가 이 게이트를 **통과(증거 채움)할 때까지** 빌드한다.
