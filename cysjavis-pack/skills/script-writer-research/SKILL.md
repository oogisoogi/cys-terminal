# script-writer-research

대본에 들어갈 모든 사실의 출처 대장을 만든다. 특히 제품 출시·성능 주장(예: "Fable 5가
무엇에 강한가")은 **반드시 1차 출처(공식 발표·문서)로 확인**한다 — 틀리면 평판이 손상된다.

## 절차

1. **클레임 후보 추출** → 검증: 대본이 말할 주장들을 목록화(성능·날짜·수치·비교·고유명사).
2. **검색 선행** → 검증: 각 클레임을 WebSearch로 조사. 학습지식만으로 단정 금지. 공식
   출처(제조사 블로그·문서·릴리스 노트) 우선, 2차 출처는 교차검증용.
3. **교차검증** → 검증: 출처 ≥2개 일치 = 공통분모로 확정. 충돌하면 양쪽 명시·보류 표기.
4. **수치·고유명사 정밀 확인** → 검증: 버전명·모델 ID·날짜·벤치마크 수치는 1차 출처
   verbatim. 어림짐작 금지.
5. **대장 작성** → 검증: 각 항목에 `[클레임] — [출처 URL] — [신뢰도: 확정/보류]`.

## 출력 계약

`facts.md` — 클레임별 출처·신뢰도 대장. 확정 항목만 대본에서 단정문으로 쓸 수 있다.
보류 항목은 대본에서 빼거나 "~로 알려져 있다" 식 약한 표현으로만. 무출처 단정은 금지.

상위 `[[script-writer]]`로 반환. 이 대장은 뒤의 `[[script-writer-factcheck]]`가 대조 기준으로 쓴다.
