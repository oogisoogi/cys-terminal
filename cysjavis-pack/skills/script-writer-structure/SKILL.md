# script-writer-structure

좋은 사실도 구조가 나쁘면 이탈한다. 2~4분 영상의 리텐션을 위한 골격을 짠다.

## 구조 원칙

- **훅(0~15초)**: 시청 이유를 즉시 준다 — 결과·반전·질문. 패키징 메시지를 여기서 건다.
- **본론(세그먼트)**: 한 세그먼트 = 한 아이디어. 세그먼트마다 시각 변화(모션그래픽 진입
  지점)를 둬 단조로움을 깬다. 개념은 말로만 설명하지 말고 "무엇을 화면에 보여줄지" 명시.
- **리텐션 비트**: 30~45초마다 작은 페이오프(예시·수치·전환)를 배치해 이탈을 막는다.
- **CTA(마지막)**: 구독·다음 행동. 영상 목적에 맞게 1개만, 자연스럽게.

## 절차

1. **분량 배분** → 검증: 목표 길이를 훅/본론/CTA에 초 단위로 배분(말하기 속도 기준).
2. **세그먼트 정의** → 검증: 각 본론 세그먼트에 [핵심 메시지 1줄 + 시각 큐 + 예상 초].
   `facts.md`의 확정 사실에만 근거.
3. **시각 큐 부착** → 검증: 세그먼트마다 화면에 띄울 것(차트·키네틱 텍스트·카드·코드 diff)을
   적는다 — 뒤의 HyperFrames 작성·`[[video-stitch]]` 합성의 입력이 된다.
4. **흐름 점검** → 검증: 세그먼트 간 전환이 논리적인가, 훅이 본론과 페이오프로 연결되는가.

## 출력 계약

`outline.md` — `[섹션, 초, 핵심메시지, 시각큐]` 표. 총합이 목표 길이에 부합. 상위
`[[script-writer]]`로 반환. 시각 큐 열은 모션그래픽 제작의 사양이 된다.
