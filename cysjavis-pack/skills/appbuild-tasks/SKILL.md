# appbuild-tasks

PRD·화면·아키텍처를 **구현 가능한 작업목록**으로 분해한다. 산출물 `.appbuild/04-tasks.md`.
각 작업은 **엔드투엔드 가치를 내는 수직 슬라이스**(트레이서 불릿)여야 한다 — cysjavis의
`[[to-issues]]`(독립적으로 잡을 수 있는 수직 슬라이스) 방법을 배선한다.

## 하위 스킬

1. `[[appbuild-tasks-slice]]` — 기능을 엔드투엔드 수직 슬라이스로 분해(트레이서 불릿).
2. `[[appbuild-tasks-order]]` — 의존 순서·크기 산정(INVEST·작게·테스트 가능).

## 절차

1. **슬라이스 분해** → 검증: `[[appbuild-tasks-slice]]` — 각 기능을 화면+데이터+로직이 함께
   도는 슬라이스로. 수평 레이어(전부 백엔드 먼저)가 아니라 수직(기능별 관통).
2. **순서·크기** → 검증: `[[appbuild-tasks-order]]` — 의존 그래프로 순서, 각 작업을 작게·
   **수용 기준(테스트 가능)**으로.
3. **추적성** → 검증: 모든 작업이 PRD 기능·화면에 추적되는가(기능 없는 작업·작업 없는 기능 0).

## 출력 계약

`.appbuild/04-tasks.md` — `[작업 / 대응 기능·화면 / 수용 기준 / 의존]` 목록. 상위 `[[appbuild]]`로
반환 → `[[appbuild-supervisor]]`가 01~04 정합을 검증. 빌드는 이 목록을 슬라이스 단위로 수행한다.
