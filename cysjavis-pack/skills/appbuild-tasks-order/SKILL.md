# appbuild-tasks-order

슬라이스에 순서와 크기를 부여하고, 각 작업을 **검증 가능**하게 만든다.

## INVEST 점검 (각 작업)

- **I**ndependent(독립) · **N**egotiable(협상가능) · **V**aluable(가치) · **E**stimable(추정가능)
  · **S**mall(작게) · **T**estable(테스트가능). 특히 **Testable**: 작업마다 **수용 기준
  (검증 가능한 조건)** 이 있어야 한다.

## 절차

1. **의존 그래프** → 검증: 작업 간 선후 의존을 만들어 순서 결정. 순환 의존 없게.
2. **크기 조정** → 검증: 너무 큰 작업은 쪼갠다(Small). 각 작업이 한 슬라이스 단위.
3. **수용 기준** → 검증: 각 작업에 테스트 가능한 수용 기준(가능하면 EARS WHEN/SHALL). "잘
   된다" 같은 검증 불가 문구 배제.

## 출력 계약

`.appbuild/04-tasks.md`의 정렬된 작업 목록 — `[작업 / 기능·화면 추적 / 수용 기준 / 의존 / 크기]`.
상위 `[[appbuild-tasks]]`로 반환. 순환 의존·검증 불가 수용 기준은 보고·수정.
