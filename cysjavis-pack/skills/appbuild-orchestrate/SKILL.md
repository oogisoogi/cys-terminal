# appbuild-orchestrate

`05-gate.md`를 **통과(증거 채움)할 때까지** 빌드한다. cysjavis 엔진을 배선한다 — master 위임·
producer≠evaluator·autopilot. 재구현이 아니다.

> 루프 핵심: **메인 오케스트레이터는 절대 문제를 직접 수정하지 않는다** — 제어·조정만. 검증은
> 만든 페인이 아닌 다른 페인이 증거로. 게이트 통과까지 루프.

## 하위 스킬 (위임→검증→라우팅 루프)

1. `[[appbuild-orchestrate-delegate]]` — master가 작업을 백/프론트 페인에 위임(키미·오퍼스 등).
2. `[[appbuild-orchestrate-verify]]` — **다른 페인**(코덱스 등)이 체크리스트로 E2E 직접 실행·증거 수집.
3. `[[appbuild-orchestrate-route]]` — 검증이 잡은 문제를 프론트/백 판별해 해당 페인에 수정 위임.

## 절차 (루프)

1. **페인 셋업** → 검증: master(오케스트레이터)·백엔드·프론트엔드·검증·예비·브라우저 페인 기동
   (`cys boot`/launch-agent). 메인은 지능 높은 모델 권장(판별 책임).
2. **위임 빌드** → 검증: `[[appbuild-orchestrate-delegate]]` — `04-tasks.md` 슬라이스를 백/프론트
   페인에 위임. 메인은 모니터링·조정만.
3. **증거 검증** → 검증: `[[appbuild-orchestrate-verify]]` — 다른 페인이 E2E로 직접 실행, 결과를
   **리포트만**(고치지 않음). `05-gate.md`의 해당 행 증거 칸을 PASS 출력으로 채운다.
4. **문제 라우팅** → 검증: FAIL이면 `[[appbuild-orchestrate-route]]` — 프론트 문제는 프론트 페인,
   백 문제는 백 페인으로 수정 위임 → 재검증. 메인은 직접 수정 안 함.
5. **게이트 루프** → 검증: A=100%·B·C·D 전부 통과(증거)할 때까지 2~4 반복. autopilot 게이트
   수렴 시 종료. 한계·load 위기면 중단·보고.

## 철학 (불변)

- **메인은 수정 안 한다** — 보고 받고, 정리해서, 해당 세션에 위임, 다시 검증 시킨다.
- **증거 없는 완료 불인정** — 게이트 행은 E2E PASS 출력이 붙어야 초록.
- **검증자≠생산자** — 만든 페인이 아닌 다른 페인이 검증(객관성).

## 출력 계약

게이트 전 행 PASS(증거 첨부) + 빌드 산출물 + `loop-state.json`. 게이트 미통과면 종료 금지·
루프 유지. 종료 보고: 게이트 통과 증거·구현 슬라이스·미해결 0.
